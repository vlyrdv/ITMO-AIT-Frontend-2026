# Лабораторная работа 1 - Вёрстка сайта средствами HTML, CSS и Bootstrap

## Цель работы
Сверстать многостраничный сайт с использованием HTML, CSS и Bootstrap, реализовать основные страницы интерфейса, продумать адаптивность и базовую интерактивность на JavaScript.

## Краткое описание проекта
В качестве основы был использован мой проект **movavifest**. Проект представляет собой веб-приложение интернет-магазинного типа с каталогом, поиском, корзиной, избранным, страницей заказов и страницей авторизации.

Стек проекта:
- Node.js
- Express
- Handlebars (hbs)
- HTML / CSS
- Bootstrap 5
- JavaScript

## Какие страницы были реализованы
В рамках вёрстки были подготовлены следующие страницы:
- главная страница
- каталог товаров
- страница поиска
- корзина
- избранное
- заказы
- страница авторизации и регистрации
- информационная страница

Это покрывает обязательные требования курса: наличие страницы входа, регистрации, пользовательской части и страницы поиска с фильтрацией.

## Использование Bootstrap
Для ускорения разработки и обеспечения базовой адаптивности был подключён Bootstrap через CDN.

Пример подключения:

```html
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/css/bootstrap.min.css" rel="stylesheet">
```

Bootstrap использовался для:
- сетки и контейнеров
- базовой типографики
- выравнивания блоков
- адаптивного поведения элементов интерфейса

## Структура шаблонов
Шаблоны страниц вынесены в папку `templates`, а повторяющиеся части интерфейса - в partials.

Например:
- `templates/index.hbs`
- `templates/catalog.hbs`
- `templates/search.hbs`
- `templates/cart.hbs`
- `templates/partials/header.hbs`
- `templates/partials/footer.hbs`

Такой подход позволил переиспользовать шапку и подвал на всех страницах.

## Пример реализации шапки
В проекте шапка сайта вынесена в отдельный partial. В ней размещены ссылки на основные разделы, а также блок входа/выхода.

Пример условного отображения авторизации:

```hbs
{{#if islogin}}
  <a href="/orders">{{islogin}}</a> - <a href="/logout">выйти</a>
{{else}}
  <a href="/auth">Войти</a>
{{/if}}
```

Это позволяет менять содержимое интерфейса в зависимости от состояния пользователя.

## Пример вёрстки каталога
Каталог товаров выводится через цикл по данным, переданным в шаблон.

```hbs
{{#if data}}
  {{#each data}}
    <div class="product-card">
      <div class="type">{{sm_desc}}</div>
      <div class="name">{{@key}}</div>
      <div class="rating">{{raiting}}</div>
      <div class="price">{{price}} р</div>
    </div>
  {{/each}}
{{else}}
  По вашему запросу ничего не найдено
{{/if}}
```

За счёт этого одна и та же структура карточек используется для разных наборов данных.

## Пользовательская интерактивность
Для ЛР1 также была продумана базовая клиентская логика:
- открытие и закрытие строки поиска
- мобильное меню
- затемнение фона при открытии overlay
- переключение состояния элементов шапки

Пример JavaScript:

```js
searchButton.addEventListener("click", () => {
  desktopNav.classList.add("hide");
  searchContainer.classList.remove("hide");
  overlay.classList.add("show");
});

menuIconContainer.addEventListener("click", () => {
  navContainer.classList.toggle("active");
});
```

## Адаптивность
Адаптивность была реализована с помощью Bootstrap и собственных media queries в CSS.

Пример:

```css
@media screen and (max-width: 850px) {
  main .home .text { display: none; }
  main .home .foto { height: 450px; width: 700px; }
}
```

Это позволяет корректно отображать интерфейс на планшетах и мобильных устройствах.
